# Storage Collections — SIKESRA (Sesuai Implementasi Nyata)

**Platform**: Cloudflare D1 di belakang abstraksi `PluginStorageConfig` milik EmDash — plugin tidak menulis SQL apa pun.
**Akses**: `ctx.storage.<collectionName>.get/put/delete/exists/getMany/putMany/deleteMany/query/count`

> Dokumen ini menggantikan versi sebelumnya yang menyebut tabel SQL `sikesra_*` dengan `CREATE TABLE IF NOT EXISTS` — pola itu **tidak ada** di kode. Lihat `03.PLUGIN_ARCHITECTURE.md` § 8 untuk daftar penyimpangan lengkap.

## 4. Enum Nyata (dengan Catatan Inkonsistensi)

### `entityType` (8 parent type, dari `DEFAULT_DATA_TYPES`)

```text
rumah_ibadah | lembaga_keagamaan | pendidikan_keagamaan | lks | guru_agama | anak_yatim | disabilitas | lansia_terlantar
```

Setiap parent type punya `subTypes` (mis. `rumah_ibadah` → Masjid, Gereja Protestan, Gereja Katolik, Pura, Wihara, Klenteng, Lainnya).

### `sensitivity`

```text
public_safe | internal | restricted | highly_restricted
```

### `verificationStage` (9 nilai, urut, dari `VERIFICATION_STAGE_FLOW`)

```text
draft → submitted_village → verified_village → submitted_district → verified_district →
submitted_sopd → verified_sopd → submitted_regency → active_verified
```

### `VerificationUserLevel` (alias `SikesraUserLevel`, dipakai di entity/event)

```text
desa_kelurahan | kecamatan | sopd | kabupaten | admin_sikesra
```

### `VerificationLevel` (dipakai untuk bucket stage→verifier, **TIDAK** sama dengan di atas)

```text
desa_kelurahan | kecamatan | sopd | kabupaten_admin | tampil
```

Pemetaan stage → level: `draft`/`submitted_village` → `desa_kelurahan`; `verified_village`/`submitted_district` → `kecamatan`; `verified_district`/`submitted_sopd` → `sopd`; `verified_sopd`/`submitted_regency` → `kabupaten_admin`; `active_verified` → `tampil`. Level `tampil` **tidak mengizinkan verifier mana pun** (`getAllowedVerifierLevels("tampil")` mengembalikan array kosong).

### `result` (verification)

```text
approved | needs_review | rejected
```

### `effect` (ABAC)

```text
allow | deny
```

## 5. Region dan Tipe Data: Statis + KV Override (Bukan Collection)

```typescript
interface AdministrativeRegion { code: string; name: string; }
interface AdministrativeDistrict extends AdministrativeRegion { villages: AdministrativeRegion[]; }
interface AdministrativeRegency extends AdministrativeRegion { districts: AdministrativeDistrict[]; }
interface AdministrativeProvince extends AdministrativeRegion { regencies: AdministrativeRegency[]; }
```

`DEFAULT_REGION_TREE: AdministrativeProvince[]` berisi data statis: 1 provinsi (Kalimantan Tengah, `62`) → 1 kabupaten (Kotawaringin Barat, `6201`) → 6 kecamatan dengan desa masing-masing. KV key `custom:regions` dapat menimpa ini lewat `regions/save` **tanpa validasi shape apa pun** — request yang salah bentuk akan tersimpan begitu saja.

Pola yang identik untuk `custom:data-types` menimpa `DEFAULT_DATA_TYPES`.

## 6. Seed Catalog Saat Install/Activate

`ensureAccessCatalogSeeded()` dan `ensureAbacCatalogSeeded()` dipanggil di hook `plugin:install` dan `plugin:activate` — mengisi `permissionCatalog`, `roleCatalog`, `abacAttributeCatalog`, `abacPolicyRules` jika collection masih kosong. Detail isi seed mengikuti struktur tipe lokal § 2.2, bukan `fixtures.ts`.

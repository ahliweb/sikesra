# Addendum: Master Data Wilayah — SIKESRA (Sesuai Implementasi Nyata)

> Versi sebelumnya dokumen ini mengusulkan tabel SQL `sikesra_region_reference`. Audit kode menemukan model wilayah nyata sudah ada dan berbeda bentuk — lihat § 1.

## 1. Model Nyata: Pohon Statis + Override KV (Bukan Tabel Referensi)

```typescript
// src/runtime.ts baris ~20-35
interface AdministrativeRegion { code: string; name: string; }
interface AdministrativeDistrict extends AdministrativeRegion { villages: AdministrativeRegion[]; }
interface AdministrativeRegency extends AdministrativeRegion { districts: AdministrativeDistrict[]; }
interface AdministrativeProvince extends AdministrativeRegion { regencies: AdministrativeRegency[]; }
```

`DEFAULT_REGION_TREE: AdministrativeProvince[]` adalah array bersarang statis di kode: 1 provinsi ("Kalimantan Tengah", `62`) → 1 kabupaten ("Kotawaringin Barat", `6201`) → 6 kecamatan, masing-masing dengan daftar desa. Route `regions/get` mengembalikan `(await ctx.kv.get("custom:regions")) ?? DEFAULT_REGION_TREE`; route `regions/save` menulis JSON apa pun yang dikirim ke key KV `custom:regions` — **tanpa validasi shape** (lihat `10.SECURITY_AND_PRIVACY_CHECKLIST.md` § 0 poin 5, item backlog H1-04).

**Tidak ada collection storage untuk wilayah** — `regions` tidak terdaftar di `AWCMS_SIKESRA_STORAGE` (`03.PLUGIN_ARCHITECTURE.md` § 3).

## 2. Bagaimana Registry Entity Merujuk Wilayah

`SikesraReferenceRegistryEntity.region` (di `src/fixtures.ts`) menyimpan `{ provinceCode, regencyCode, districtCode, villageCode }` sebagai field bebas — **tidak ada validasi terhadap `DEFAULT_REGION_TREE`/KV override saat entity disimpan**. Seseorang dapat menulis kode wilayah yang tidak ada di pohon tanpa ditolak.

## 3. Gap dan Rekomendasi Hardening (Bukan Fitur Baru — Bagian dari H1-04)

```typescript
// Pola validasi yang BELUM ada di kode, direkomendasikan sebagai bagian H1-04
async function validateRegionAgainstTree(ctx: PluginContext, region: {
  provinceCode: string; regencyCode: string; districtCode: string; villageCode: string;
}) {
  const tree = (await ctx.kv.get<AdministrativeProvince[]>("custom:regions")) ?? DEFAULT_REGION_TREE;
  const province = tree.find((p) => p.code === region.provinceCode);
  const regency = province?.regencies.find((r) => r.code === region.regencyCode);
  const district = regency?.districts.find((d) => d.code === region.districtCode);
  const village = district?.villages.find((v) => v.code === region.villageCode);
  if (!province || !regency || !district || !village) {
    throw new Error("SIKESRA_REGION_HIERARCHY_INVALID");
  }
}
```

Ini adalah **rekomendasi untuk H1-04**, bukan kode yang sudah ada. Jangan asumsikan validasi ini berjalan sampai item backlog ini benar-benar dikerjakan dan diverifikasi dengan test.

## 4. Mengubah Data Wilayah Saat Ini

Satu-satunya cara mengubah data wilayah hari ini adalah memanggil `regions/save` dengan payload `AdministrativeProvince[]` lengkap (menimpa seluruhnya, bukan menambah satu desa) — tidak ada operasi tambah/ubah/hapus granular per desa/kecamatan di level API saat ini. Ini adalah keterbatasan nyata, bukan gap keamanan.

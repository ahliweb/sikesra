## 1. Cara Menggunakan Index Ini

### Untuk AI Agent (Junior Model)

1. Baca issue GitHub yang akan dikerjakan
2. Lihat label `epic:*` untuk tahu EPIC mana
3. Baca dokumen yang **dikutip di issue** saja (biasanya 1–2 dokumen)
4. Load skill yang disebutkan di issue
5. JANGAN baca semua dokumen di index ini — terlalu banyak token

### Untuk Developer Manusia

1. Mulai dari PRD utama untuk gambaran besar
2. Lanjut ke dokumen teknis yang relevan dengan fitur yang dikerjakan
3. Cek Sprint Plan untuk urutan implementasi
4. Gunakan API Contract dan DB Schema sebagai referensi aktif

### Untuk Reviewer / Auditor

1. PRD → Architecture → DB Schema → API Contract
2. Security Checklist untuk review keamanan
3. UAT Checklist untuk verifikasi deployment

## 6. Rekomendasi Proses Berikutnya

### Prioritas MVP (Sprint 0–2)

1. **Sprint 0**: Plugin scaffold + D1 migrations (EPIC-01-001..004)
   - Paling kritis — semua epic lain bergantung pada ini
   - Buat issue #[N+1] untuk S0-01..S0-06

2. **Sprint 1**: UI/UX Foundation (EPIC-00-UX-01..05 + EPIC-07-007-01..02)
   - Blockers: komponen shared dibutuhkan semua halaman lain
   - Buat issue per UX item (atomic, per komponen)

3. **Sprint 2**: Registry + Documents (EPIC-02 + EPIC-04)
   - Fitur core yang paling visible ke pengguna

### Setelah Sprint 2 Selesai

- Demo ke stakeholder → feedback loop
- Evaluasi apakah perlu adjustment di Sprint 3 (verification)
- Pertimbangkan pilot dengan 1 kecamatan

### Integrasi Berikutnya (Post-MVP)

- Export data → DTKS / SATUDATA (butuh API spec dari Kemensos)
- Notifikasi → perlu email provider (Resend / Cloudflare Email Workers)
- Mobile → Progressive Web App (PWA) dari template yang ada

**Status**: Active
**Update**: Juni 2026 — ditulis ulang total setelah audit kode menemukan bahwa fitur registry/verifikasi/dokumen/ABAC/akses/audit **sudah terimplementasi** (39 route, 16 halaman admin). Versi sebelumnya dari dokumen ini berasumsi "MVP belum dikerjakan" — itu salah. Lihat `03.PLUGIN_ARCHITECTURE.md` § 8 dan `10.SECURITY_AND_PRIVACY_CHECKLIST.md` untuk dasar temuan.

**Prinsip backlog baru**: fitur fungsional sudah ada. Gap nyata adalah **otorisasi server-side, konsistensi tipe internal, dan validasi input** — bukan UI atau route yang belum dibuat. Item di bawah adalah gap yang ditemukan lewat audit kode langsung, bukan asumsi.

## EPIC-H1 — Otorisasi Server-Side Nyata (Prioritas Tertinggi)

### H1-01 Identitas Terverifikasi pada Route Mutasi

**Gap**: `getRequestUserId()` membaca header `X-Sikesra-User-Id` yang tidak ditandatangani; `verification/advance`/`verification/reject` membaca `verifierLevel`/`actor` langsung dari body request milik pemanggil. Tidak ada satu route mutasi pun yang memverifikasi identitas pemanggil terhadap sesi/auth EmDash yang sah.

**Pekerjaan**: ganti sumber identitas dari header tak-ditandatangani/body request menjadi identitas dari sesi EmDash yang terverifikasi (`ctx.user`/`locals.user` atau API setara yang disediakan framework).

### H1-02 Permission Check pada Semua Route `*/save`

**Gap**: `registry/save`, `documents/save`, `import/promote`, `settings/save`, seluruh `access/*/save`, seluruh `abac/*/save`, `regions/save`, `data-types/save` — tidak ada satu pun yang memeriksa permission sebelum menulis.

**Pekerjaan**: tambahkan pengecekan permission di setiap route mutasi, bersumber dari `permissionCatalog`/`roleCatalog`/`userRoleAssignments` (collection yang sudah ada) — bukan dari `src/permissions.ts` yang stringnya sudah tidak sinkron dengan manifest (lihat H2-02).

### H1-03 ABAC Sebagai Gerbang Nyata, Bukan Simulasi

**Gap**: `evaluateAbacDecision()` hanya dipanggil dari `abac/preview` dan `abac/enforce-demo` (yang sendiri tidak pernah memblokir). Tidak ada mutasi nyata yang melewati evaluasi ABAC.

**Pekerjaan**: panggil `evaluateAbacDecision()` (atau refactor-nya) di `registry/save`, `verification/advance`, `verification/reject`, `documents/save` sebelum menulis, dan benar-benar menolak (return error) jika hasilnya `deny`.

### H1-04 Validasi Shape pada `regions/save` dan `data-types/save`

**Gap**: kedua route menulis JSON apa pun yang dikirim ke KV tanpa validasi sama sekali.

**Pekerjaan**: tambahkan validasi Zod terhadap shape `AdministrativeProvince[]` / `SikesraParentType[]` sebelum menyimpan ke KV.

## EPIC-H2 — Konsistensi Tipe dan Nama Internal

### H2-01 Rekonsiliasi `VerificationLevel` vs `VerificationUserLevel`

**Gap**: dua union type yang tidak sinkron (`kabupaten_admin`/`tampil` vs `kabupaten`/`admin_sikesra`) dijembatani manual di `getAllowedVerifierLevels()`. Level `"tampil"` tidak mengizinkan verifier mana pun — kemungkinan unintentional dead-end untuk stage `active_verified`.

**Pekerjaan**: putuskan satu union type kanonik (rekomendasi: pakai `VerificationUserLevel`/`SikesraUserLevel` di semua tempat), refactor `getVerificationLevel()`/`getAllowedVerifierLevels()` mengikuti, dan tentukan apakah `active_verified` seharusnya memang tidak punya verifier lanjutan (final state) — dokumentasikan keputusan ini di `12.CHANGE_CONTROL_AND_DECISION_LOG.md`.

### H2-02 Sinkronkan `src/permissions.ts` dengan String Permission Nyata

**Gap**: `AWCMS_SIKESRA_PERMISSIONS.accessPreviewRead = "awcms:sikesra:access-preview:read"` tidak cocok dengan string nyata di manifest `runtime.ts` (`"awcms:sikesra:preview:read"`). File ini hanya dipakai test, tidak dipakai runtime — berisiko menyesatkan kontributor baru yang membacanya sebagai sumber kebenaran.

**Pekerjaan**: audit semua string permission di manifest `runtime.ts`, samakan dengan `permissions.ts`, atau hapus `permissions.ts` jika digantikan sepenuhnya oleh `permissionCatalog` collection yang di-seed dinamis.

### H2-03 Hapus atau Sambungkan `src/audit.ts`

**Gap**: dead code, definisi duplikat `createAuditRecord`/`ExampleAuditEvent` yang membingungkan dibanding versi nyata di `runtime.ts`.

**Pekerjaan**: hapus file ini, atau jika ada rencana memisahkan modul audit dari `runtime.ts` (mengurangi ukuran file 2870 baris), migrasikan implementasi nyata ke sana dan update semua import.

### H2-04 Rename Sisa Nama Generik Scaffold (`Example*`)

**Gap**: `ExampleAuditEvent`, `ExampleSettings`, copy hook `"Installed the AWCMS-Micro example plugin"` adalah sisa nama scaffold generik, belum di-rename ke istilah SIKESRA.

**Pekerjaan**: rename bertahap (non-breaking — tipe internal, tidak ada di response envelope publik) sebagai PR terpisah, kecil, mudah direview.

## EPIC-H3 — Test Coverage untuk Fitur yang Sudah Ada

### H3-01 Test Level-Route untuk Registry CRUD

**Gap**: `tests/plugin.test.ts` menguji descriptor/manifest/fixtures, bukan perilaku route `registry/save`/`registry/list`.

**Pekerjaan**: tambah test yang memanggil route handler langsung, menguji create/list/idempotency.

### H3-02 Test Verification Stage Machine

**Pekerjaan**: test `getNextVerificationStage()`, `getRevisionTargetStage()`, dan `verification/advance`/`verification/reject` end-to-end untuk semua 9 stage, termasuk kasus tepi `active_verified` (level `tampil`, tidak ada verifier — pastikan perilaku ini disengaja, lihat H2-01).

### H3-03 Test ABAC Evaluation

**Pekerjaan**: test `evaluateAbacDecision()` dengan kombinasi subject/resource/context yang representatif, termasuk default-deny saat tidak ada policy yang cocok.

### H3-04 Security Regression Test untuk Gap H1

**Pekerjaan**: setelah H1-01..H1-04 dikerjakan, tambahkan test regresi yang membuktikan: request tanpa identitas sah ditolak, request dengan permission salah ditolak, ABAC deny benar-benar menolak mutasi.

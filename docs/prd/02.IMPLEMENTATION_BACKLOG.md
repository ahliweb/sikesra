## EPIC-02 — Registry Management (Sprint 2)

### 002-01 CRUD Entitas Dasar
- `POST /registry` — buat entitas baru (stage: draft)
- `GET /registry` — list dengan filter + cursor pagination
- `GET /registry/:id` — detail
- `PUT /registry/:id` — update (hanya owner level atau atas)
- `DELETE /registry/:id` — soft delete (sensitivity: set deleted)
- Audit log setiap operasi

### 002-02 Form Wizard Registry
- Step 1: Info Dasar (label, tipe entitas)
- Step 2: Wilayah (RegionSelector: provinsi → desa)
- Step 3: Detail (sensitivity, public summary, atribut dinamis per tipe)
- Step 4: Dokumen (upload opsional)
- Save draft di setiap step (localStorage state)

### 002-03 Filter & Paginasi
- Filter: tipe entitas (multi-select), wilayah, stage, sensitivity
- Cursor-based pagination (limit 50, max 100)
- Sort: created_at DESC (default)
- ABAC: otomatis filter berdasarkan level user

### 002-04 Tipe Entitas Dinamis
- Default 8 tipe dari `DEFAULT_DATA_TYPES` di runtime.ts
- Admin bisa tambah tipe baru via KV (`custom:data-types`)
- Setiap tipe: id, label, description, icon, color, extraFields (schema)

## EPIC-03 — Verification Workflow (Sprint 3)

### 003-01 Verification Queue
- `GET /registry/verification-queue` — list entitas menunggu verifikasi
- Filter otomatis: hanya entitas di level berikutnya dari level user
- Sort: submitted_at ASC (FIFO)

### 003-02 Verify Action
- `POST /registry/:id/verify` — body: `{ result, notes }`
- Validasi: user harus di level yang tepat untuk stage ini
- Transisi stage sesuai alur (approve / needs_review / reject)
- Catat verification_event + audit_event

### 003-03 Verification History
- `GET /registry/:id/verifications` — list verification events
- Timeline per entitas: stage, actor, result, notes, timestamp

### 003-04 Needs Revision Flow
- `needs_review`: entitas kembali ke stage sebelumnya
- Notifikasi di navigation badge untuk penginput
- Notes wajib diisi jika `needs_review` atau `rejected`

## EPIC-04 — Documents (Sprint 2)

### 004-01 Upload Dokumen
- `POST /documents` — multipart/form-data
- Validasi: max 10MB, tipe: PDF, JPG, PNG
- Upload ke R2 dengan prefix `sikesra/documents/{entityId}/`
- Simpan metadata di `sikesra_supporting_documents`

### 004-02 List & View Dokumen
- `GET /documents?entityId=` — list per entitas
- `GET /documents/:id/download` — signed URL (5 menit) dari R2
- Filter sensitivity (tidak tampilkan `highly_restricted` ke level rendah)

## EPIC-05 — ABAC Engine (Sprint 4)

### 005-01 Attribute Catalog
- CRUD attribute catalog: `POST/GET/PUT/DELETE /abac/attributes`
- Attribute default: `user.level`, `user.district_code`, `user.village_code`, `resource.entity_type`, `resource.village_code`, `resource.sensitivity`

### 005-02 Policy Rules
- CRUD policy rules: `POST/GET/PUT/DELETE /abac/policies`
- Fields: label, effect (allow/deny), actions, required_subject, required_resource, required_context
- Validasi: attribute keys harus ada di catalog

### 005-03 ABAC Preview
- `POST /abac/preview` — evaluate tanpa enforce
- Input: subject attributes, resource attributes, context, action
- Output: result (allow/deny), matched_policy?, evaluation_trace

### 005-04 Region Filter Middleware
- Helper `buildRegionFilter(user)` → `{ village_code?, district_code?, regency_code? }`
- Diterapkan di setiap query entitas
- `admin_sikesra` dan `kabupaten`: tidak ada filter wilayah

## EPIC-06 — Audit & Reporting (Sprint 4)

### 006-01 Audit Log UI
- Timeline view: kind icon, summary, actor, timestamp
- Filter: kind prefix, actor, date range
- Paginasi cursor
- Export: JSON download (admin only)

### 006-02 Reports Dashboard
- Bar chart: total + verified per tipe entitas
- Filter wilayah (sesuai level user)
- Aggregate per kecamatan (kabupaten view)
- Suppressed categories dengan label "(Data dirahasiakan)"

### 006-03 Public Status Endpoint
- `GET /public/status` — tanpa auth
- Hanya aggregate: `{ categories: [{code, label, total, verified, suppressed}], caveat }`
- Tidak ada data per-record, tidak ada PII
- `small_cell_threshold`: jika count < threshold → suppressed=true

### 006-04 Audit Retention Cron
- Cron hook: delete events older than `retention_days` (default 90)
- Log: `[sikesra:cron] Deleted N audit events older than 90 days`

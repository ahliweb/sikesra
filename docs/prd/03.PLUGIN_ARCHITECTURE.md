# Plugin Architecture — SIKESRA v1.0

**Plugin**: `@ahliweb/awcms-sikesra`  
**Plugin ID**: `awcms-micro-sikesra`  
**Format**: Native EmDash Plugin  

---

## 1. Prinsip Arsitektur Plugin

### 1.1 Plugin-Owned Sepenuhnya

Plugin SIKESRA tidak boleh:
- Mengubah tabel core EmDash (`_emdash_*`)
- Memodifikasi auth/session middleware core
- Menyimpan data di namespace lain

Plugin SIKESRA boleh:
- Membuat tabel sendiri (`sikesra_*`) via install hook
- Mendaftarkan routes di prefix `/_emdash/api/plugins/awcms-sikesra/`
- Mendaftarkan admin pages + widgets via manifest
- Menulis ke storage namespace sendiri
- Memasang hooks pada lifecycle plugin + content events

### 1.2 Native vs Sandboxed

**Native entry** (`src/index.ts`): digunakan saat plugin dipasang di project lokal (trusted).
- Akses penuh ke EmDash internals via `definePlugin()`
- Admin pages didaftarkan via `PluginDescriptor.adminPages`

**Sandboxed entry** (`src/sandbox.ts`): untuk deployment terpisah (future marketplace).
- Terbatas pada public API
- Tidak akses internals

### 1.3 Separation of Concerns

| File | Tanggung Jawab |
|------|---------------|
| `src/index.ts` | Plugin descriptor + entrypoints |
| `src/runtime.ts` | Storage, routes, hooks, manifest |
| `src/admin.tsx` | Admin UI (React + Kumo) |
| `src/navigation.ts` | Navigation models + adapter |
| `src/permissions.ts` | Permission constants |
| `src/audit.ts` | Audit record helper |
| `src/fixtures.ts` | Reference data + TypeScript types |
| `src/sandbox.ts` | Sandboxed server entry |

## 3. Storage Schema (Plugin-Owned D1 Tables)

Semua tabel menggunakan prefix `sikesra_`. Dibuat di `install` hook, idempoten.

### Core Registry Tables

```sql
-- Entitas utama
CREATE TABLE IF NOT EXISTS sikesra_registry_entities (
  id TEXT PRIMARY KEY,
  code TEXT UNIQUE NOT NULL,
  label TEXT NOT NULL,
  entity_type TEXT NOT NULL,
  sensitivity TEXT NOT NULL DEFAULT 'internal',
  province_code TEXT NOT NULL,
  regency_code TEXT NOT NULL,
  district_code TEXT NOT NULL,
  village_code TEXT NOT NULL,
  verification_stage TEXT NOT NULL DEFAULT 'draft',
  input_level TEXT NOT NULL DEFAULT 'desa_kelurahan',
  public_summary TEXT,
  extra_data TEXT,  -- JSON untuk atribut dinamis per tipe
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))

CREATE INDEX IF NOT EXISTS idx_sre_code ON sikesra_registry_entities(code);
CREATE INDEX IF NOT EXISTS idx_sre_entity_type ON sikesra_registry_entities(entity_type);

CREATE TABLE IF NOT EXISTS sikesra_verification_events (

## 4. Route Architecture

```

├── GET  /overview                   → dashboard:read

## 5. Hook Surface

```typescript
// install: buat semua tabel, seed default roles/permissions
// activate: validasi storage, reset counters
// deactivate: bersihkan cron
// uninstall: hapus semua data (konfirmasi required)

// content:afterCreate / afterUpdate: snapshot untuk audit
// media:afterUpload: log R2 key

// cron:daily: audit retention cleanup (delete events > retention_days)
// cron:weekly: orphan document cleanup

// page:metadata: inject SIKESRA meta ke halaman publik jika enabled
```

## 7. ABAC Engine Design

```typescript
interface AbacEvaluationInput {
  action: string
  subject: Record<string, string>  // user attributes
  resource: Record<string, string>  // entity attributes
  context?: Record<string, string>  // request context
}

interface AbacEvaluationResult {
  result: "allow" | "deny"
  matchedPolicy?: string
  evaluationTrace: Array<{ policyId: string, effect: string, matched: boolean }>
}

// Evaluation order:
// 1. Collect all policies
// 2. Sort by priority DESC
// 3. For each policy: check if subject+resource+context match required attributes
// 4. First match wins (first matched policy determines result)
// 5. If no policy matches: default DENY
```

## 8. Error Codes

```typescript
// Plugin-specific error codes
"SIKESRA_NOT_INITIALIZED"     // Plugin belum di-setup
"SIKESRA_ENTITY_NOT_FOUND"    // Entitas tidak ditemukan
"SIKESRA_STAGE_INVALID"       // Transisi stage tidak valid
"SIKESRA_LEVEL_MISMATCH"      // User level tidak cocok untuk verifikasi stage ini
"SIKESRA_REGION_FORBIDDEN"    // User tidak bisa akses wilayah ini (ABAC)
"SIKESRA_SENSITIVITY_DENIED"  // Sensitivity level terlalu tinggi untuk user
"SIKESRA_DOCUMENT_TOO_LARGE"  // File > 10MB
"SIKESRA_DOCUMENT_TYPE_INVALID" // Tipe file tidak didukung
```

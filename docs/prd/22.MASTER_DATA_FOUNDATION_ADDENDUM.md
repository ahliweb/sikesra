> Versi sebelumnya dokumen ini mengusulkan tabel `sikesra_org_units`, `sikesra_document_numbering_sequences`, `sikesra_holiday_calendar` yang tidak ada di kode dan tidak punya dasar di backlog nyata. Ditulis ulang untuk mendeskripsikan model tipe data entitas yang sebenarnya ada.

## 1. Model Nyata: Parent/Sub-Type Statis + Override KV

```typescript
// src/runtime.ts baris ~182-194
interface SikesraSubType { code: string; label: string; }
interface SikesraParentType { id: string; code: string; label: string; subTypes: SikesraSubType[]; }
```

`DEFAULT_DATA_TYPES: SikesraParentType[]` berisi tepat 8 parent type, masing-masing dengan sub-type:

| Parent (`id`) | Jumlah Sub-Type | Contoh Sub-Type |

| `rumah_ibadah` | 7 | Masjid, Gereja Protestan, Gereja Katolik, Pura, Wihara, Klenteng, Lainnya |

Route `data-types/get` mengembalikan `(await ctx.kv.get("custom:data-types")) ?? DEFAULT_DATA_TYPES`; `data-types/save` menulis JSON apa pun ke KV `custom:data-types` **tanpa validasi shape** (sama seperti region, lihat `21.ADDENDUM_MASTER_DATA_WILAYAH.md` dan backlog H1-04).

**Tidak ada collection storage untuk tipe data** ΓÇË `dataTypes` tidak terdaftar di `AWCMS_SIKESRA_STORAGE`.

## 2. Apa yang TIDAK Ada (Dihapus dari Versi Sebelumnya Dokumen Ini)

Tidak ditemukan di kode dan dihapus dari dokumen ini karena tidak berdasar:

- Tabel unit organisasi SOPD (`sikesra_org_units`) ΓÇË routing antrian verifikasi ke SOPD yang tepat saat ini didasarkan pada `VerificationLevel`/stage, bukan pemetaan SOPDΓ¶Ëentity-type
- Tabel penomoran dokumen resmi (`sikesra_document_numbering_sequences`) ΓÇË tidak ada fitur penerbitan surat bernomor di kode
- Tabel kalender hari libur (`sikesra_holiday_calendar`) ΓÇË tidak ada perhitungan SLA berbasis hari kerja di kode

Jika fitur-fitur ini memang dibutuhkan ke depan, ajukan lewat Discussion (`12.CHANGE_CONTROL_AND_DECISION_LOG.md` ┬¯ 2) sebagai fitur baru yang jelas scope-nya ΓÇË jangan asumsikan sudah ada dasar implementasinya.

## 3. Mengubah Tipe Data Saat Ini

Sama seperti wilayah: satu-satunya cara mengubah katalog tipe data adalah memanggil `data-types/save` dengan payload `SikesraParentType[]` lengkap (menimpa seluruhnya), tidak ada operasi granular per tipe.

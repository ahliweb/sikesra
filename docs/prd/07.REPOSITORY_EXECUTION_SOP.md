# Repository Execution SOP — SIKESRA

**Untuk**: Developer (AI agent dan manusia)  
**Tujuan**: SOP eksekusi kerja di repo ini tanpa harus membaca semua dokumen

---

## 1. Sebelum Mulai (Wajib)

```bash
# 1. Cek status repo
git status && git log --oneline -5

# 2. Pastikan lint bersih (baseline sebelum kamu)
pnpm --silent lint:json | jq '.diagnostics | length'
# Jika hasilnya > 0: FIX dulu sebelum mulai

# 3. Load skill yang relevan (lihat tabel di bawah)
# Baca file SKILL.md yang relevan sebelum koding
```

### Skill yang Dimuat per Tipe Kerja

| Tipe Pekerjaan | Skill File |
|---------------|------------|
| Plugin scaffold / extend / hooks | `skills/sikesra-plugin-execution/SKILL.md` |
| Migrasi D1 / tabel / storage | `skills/sikesra-data-d1/SKILL.md` |
| API routes / RBAC / ABAC | `skills/sikesra-api-rbac/SKILL.md` |
| Admin UI / komponen React | `skills/sikesra-ui-admin/SKILL.md` |
| Plugin upstream (EmDash internals) | `awcmsmicro-dev/skills/creating-plugins/SKILL.md` |
| Site/template Astro | `awcmsmicro-dev/skills/building-emdash-site/SKILL.md` |

## 2. Selama Kerja

```bash
# Setiap selesai edit file:
pnpm --silent lint:quick

# Setelah edit TypeScript/TSX:
pnpm --filter @ahliweb/awcms-sikesra typecheck

# Format sebelum commit:
pnpm format

# Build plugin (cek output):
pnpm --filter @ahliweb/awcms-sikesra build

# Run tests:
pnpm --filter @ahliweb/awcms-sikesra test
```

## 3. Alur Kerja Issue

### Untuk setiap GitHub Issue:

1. **Baca issue** → perhatikan Context Capsule di atas issue
2. **Load skill** yang disebutkan di issue
3. **Baca "References"** — maksimal 1-2 section dokumen + 1 skill
4. **Scope**: kerjakan HANYA yang ada di "Scope" issue
5. **Jangan** modifikasi file di luar scope issue
6. **Tulis test** sebelum implementasi jika ada bug
7. **Cek DoD** di issue sebelum selesai

## 5. Naming Conventions

| Entity | Convention |
|--------|------------|
| Plugin npm | `@ahliweb/awcms-sikesra` |
| Plugin ID | `awcms-sikesra` |
| Template | `awcms-sikesraTemplate` atau `awcms-sikesraTemplate-cloudflare` |
| Storage collections | `camelCase`, lihat `04.DATABASE_SCHEMA.md` (BUKAN tabel SQL `sikesra_*`) |
| KV keys | `custom:regions`, `custom:data-types` (lihat `03.PLUGIN_ARCHITECTURE.md` § 6) |
| API prefix | `/_emdash/api/plugins/awcms-sikesra/` |
| Permission prefix | `awcms:sikesra:` |
| Audit kind | `domain.action` (e.g. `registry.created`) |

## 6. Commit Convention

```bash
# Format: type(scope): message
git commit -m "feat(registry): add CRUD endpoints for registry entities"
git commit -m "fix(verification): correct stage transition for needs_review"
git commit -m "ui(admin): add EntityTypePill component with Kumo tokens"
git commit -m "test(abac): add policy evaluation tests"
git commit -m "docs(prd): update API contract with verify endpoint"

# Wajib tambah changeset jika mengubah behavior plugin
pnpm changeset --empty
# Edit .changeset/*.md sesuai format
```

---

## 7. Changeset Format

```markdown
---
"@ahliweb/awcms-sikesra": patch
---

Adds CRUD endpoints for registry entities with ABAC filtering.
```

Bump types:
- `patch`: bug fix, doc update, test
- `minor`: fitur baru (backwards-compatible)
- `major`: breaking change (diskusikan dulu)

## 8. PR Rules

- Baca `.github/PULL_REQUEST_TEMPLATE.md` sebelum buat PR
- Semua CI check harus pass
- Centang AI disclosure jika ada AI-generated code
- Jangan spray changes — hanya file yang relevan dengan PR
- Adversarial review sebelum PR (load `skills/adversarial-reviewer/SKILL.md`)

## 9. Troubleshooting

### Build error setelah sync upstream

```bash
# Cek diff dari upstream
pnpm sync:dry-run

# Validasi protected paths
pnpm validate:boundaries

# Re-sync jika perlu
pnpm sync:upstream
```

### TypeScript error di admin.tsx

```bash
# Cek Kumo component API
npx @cloudflare/kumo doc Button
npx @cloudflare/kumo ls

# Pastikan import dari @cloudflare/kumo (bukan custom)
```

### Storage collection bermasalah di lokal

```bash
# Plugin ini TIDAK punya file migrasi SQL — collection storage dibuat
# otomatis oleh hook plugin:install/plugin:activate di runtime.ts.
# Untuk reset state lokal saat dev:
rm -rf .wrangler/
```

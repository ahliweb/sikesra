# Training and Rollout Plan — SIKESRA

## 1. Strategi Pelatihan

### 1.1 Pendekatan Cascade Training

Admin SIKESRA kabupaten dilatih terlebih dahulu, lalu melatih verifikator SOPD/kecamatan, yang kemudian melatih petugas desa/kelurahan di wilayah masing-masing. Pendekatan berjenjang ini mengikuti struktur verifikasi SIKESRA itu sendiri.

### 1.2 Format Pelatihan

- Sesi tatap muka/virtual singkat (1-2 jam) per kelompok peran.
- Materi tertulis (SOP, `11.USER_MANUAL_AND_SOP.md`) sebagai rujukan mandiri.
- Sesi praktik langsung di environment staging sebelum go-live produksi.

### 1.3 Prinsip Pelatihan

- Latih berdasarkan peran (role-based), bukan satu sesi umum untuk semua orang — petugas desa tidak perlu tahu detail ABAC/RBAC, cukup SOP input data mereka.
- Gunakan data uji di lingkungan staging, bukan data produksi, selama pelatihan.

## 2. Segmen Pelatihan per Role

| Segmen | Peserta | Fokus Materi |
| --- | --- | --- |
| Admin Kabupaten | Admin SIKESRA, verifikator kabupaten | Kelola akses, ABAC, pengaturan, audit |
| Verifikator SOPD/Kecamatan | Petugas verifikasi level menengah | SOP verifikasi § 4-5 di `11.USER_MANUAL_AND_SOP.md` |
| Petugas Desa/Kelurahan | Petugas input data | SOP input dan pengajuan verifikasi § 3 |
| Auditor | Pihak pengawas internal | Akses audit log read-only |

## 3. Jadwal Rollout (Indikatif, Disesuaikan dengan Sprint)

Rollout dimulai setelah kriteria go-live MVP (`06.UAT_AND_DEPLOYMENT_CHECKLIST.md` § 4) terpenuhi. Urutan: pelatihan admin kabupaten → pilot terbatas (1-2 desa/kecamatan) → ekspansi bertahap ke seluruh wilayah kabupaten.

## 4. Strategi Rollout Bertahap

### 4.1 Batch 1 — Pilot Terbatas

1-2 desa/kelurahan dan kecamatan terkait menggunakan sistem secara penuh selama periode pilot untuk menemukan masalah praktis sebelum ekspansi.

### 4.2 Batch 2 — Ekspansi Bertahap

Setelah pilot stabil (tidak ada bug Blocker/Major terbuka selama minimal satu siklus verifikasi penuh), perluas ke desa/kecamatan lain secara bertahap per kecamatan.

### 4.3 Kriteria Naik ke Batch Berikutnya

- Tidak ada bug Blocker terbuka selama periode pilot.
- Petugas pilot dapat menyelesaikan siklus input → verifikasi tanpa bantuan teknis langsung.
- Feedback pilot sudah ditindaklanjuti (perbaikan UI/copy minor).

## 5. Materi Pelatihan

### 5.1 Dokumen SOP

`11.USER_MANUAL_AND_SOP.md` sebagai rujukan utama, dipecah per segmen role saat pelatihan.

### 5.2 Quick Reference

Kartu referensi cepat 1 halaman per role berisi: cara login, langkah utama SOP role tersebut, dan kontak dukungan — dibuat menjelang rollout, bukan bagian dari PRD ini.

## 6. Evaluasi Pelatihan

### 6.1 Observasi Pasca-Pelatihan

Pantau penggunaan sistem oleh peserta pilot selama 1-2 minggu pertama; catat pertanyaan berulang sebagai indikasi materi pelatihan/SOP yang perlu diperjelas.

### 6.2 Survei Kepuasan Sederhana

Survei singkat pasca-pelatihan untuk menilai kejelasan SOP dan kemudahan penggunaan UI — hasil menjadi input perbaikan `11.USER_MANUAL_AND_SOP.md`.

## 7. Dukungan Pasca-Pelatihan

### 7.1 Kanal Dukungan

Dukungan teknis internal kabupaten (kontak resmi, tidak dipublikasikan di dokumen PRD).

### 7.2 Eskalasi

Masalah yang tidak terselesaikan di level desa/kecamatan dieskalasikan ke admin SIKESRA kabupaten, yang dapat membuka isu teknis ke tim pengembang jika diperlukan.

# Sprint Execution Plan — SIKESRA Hardening

> **Catatan akurasi (Juni 2026)**: Versi sebelumnya dokumen ini merencanakan "Sprint 0-4" untuk membangun plugin dari nol. Audit kode menemukan fitur-fitur tersebut **sudah ada**. Rencana di bawah ditulis ulang mengikuti backlog hardening nyata di `02.IMPLEMENTATION_BACKLOG.md` (EPIC-H1/H2/H3).

**Total Sprint**: 3 (mengikuti 3 EPIC hardening)
**Durasi**: fleksibel per ukuran EPIC, bukan blok 2 minggu tetap — setiap item EPIC-H1/H2/H3 berdiri sendiri dan dapat dikerjakan begitu siap, tidak perlu menunggu sprint sebelumnya selesai kecuali ada dependency eksplisit (lihat dependency graph di `02.IMPLEMENTATION_BACKLOG.md`).

## Sprint H1 — Otorisasi Server-Side Nyata (Prioritas Tertinggi)

### Tujuan

Menutup gap kritis: route mutasi plugin saat ini tidak memeriksa identitas atau permission sama sekali (lihat `10.SECURITY_AND_PRIVACY_CHECKLIST.md` § 0).

### Items

| ID | Task | Dependency |
| --- | --- | --- |
| H1-01 | Identitas terverifikasi pada route mutasi (ganti header/body tak-ditandatangani) | — |
| H1-02 | Permission check pada semua route `*/save` | H1-01 |
| H1-03 | ABAC sebagai gerbang nyata (sambungkan `evaluateAbacDecision()` ke mutasi) | H1-01, H1-02 |
| H1-04 | Validasi shape Zod pada `regions/save`/`data-types/save` | — |

### Exit Criteria

- [ ] Request tanpa identitas sah ke route `*/save` ditolak
- [ ] Request dengan permission yang salah ditolak
- [ ] ABAC `deny` benar-benar menghentikan mutasi (bukan hanya tercatat log)
- [ ] `regions/save`/`data-types/save` menolak payload yang tidak sesuai shape
- [ ] Tidak ada regresi pada fitur yang sudah berfungsi (lihat `06.UAT_AND_DEPLOYMENT_CHECKLIST.md` § 3.5)

## Sprint H2 — Konsistensi Tipe dan Nama Internal

### Tujuan

Menutup inkonsistensi internal yang ditemukan audit (`03.PLUGIN_ARCHITECTURE.md` § 8) sebelum gap ini menjalar ke kode baru.

### Items

| ID | Task | Dependency |
| --- | --- | --- |
| H2-01 | Rekonsiliasi `VerificationLevel` vs `VerificationUserLevel` | — (independen dari H1) |
| H2-02 | Sinkronkan `src/permissions.ts` dengan string permission nyata di manifest | — |
| H2-03 | Hapus atau sambungkan `src/audit.ts` (dead code) | — |
| H2-04 | Rename bertahap sisa nama generik scaffold (`Example*`) | H2-01..03 selesai (hindari konflik rename bersamaan) |

### Exit Criteria

- [ ] Satu union type kanonik untuk level verifikasi, didokumentasikan di `12.CHANGE_CONTROL_AND_DECISION_LOG.md`
- [ ] `src/permissions.ts` cocok 100% dengan string di manifest, atau dihapus dan diganti `permissionCatalog` dinamis
- [ ] `src/audit.ts` dihapus atau jadi satu-satunya sumber kebenaran audit
- [ ] Tidak ada regresi typecheck/test

## Sprint H3 — Test Coverage untuk Fitur yang Sudah Ada

### Tujuan

Plugin punya fitur substansial tapi `tests/plugin.test.ts` hanya menguji descriptor/manifest, bukan perilaku route.

### Items

| ID | Task | Dependency |
| --- | --- | --- |
| H3-01 | Test level-route untuk registry CRUD | — |
| H3-02 | Test verification stage machine (9 stage + edge case `tampil`) | Sebaiknya setelah H2-01 agar tidak menguji perilaku yang akan diubah |
| H3-03 | Test ABAC evaluation (default-deny, priority matching) | — |
| H3-04 | Security regression test untuk gap H1 | H1-01..04 selesai |

### Exit Criteria

- [ ] Coverage route-level untuk registry, verification, ABAC bertambah signifikan dari baseline saat ini (descriptor-only)
- [ ] H3-04 membuktikan H1 benar-benar menutup gap (test gagal sebelum H1, lulus sesudah)

## Urutan yang Disarankan

```text
Mulai paralel: H1-01..04, H2-01..03, H3-01..03 (semua independen satu sama lain)
H2-04 (rename)        → setelah H2-01..03 selesai
H3-04 (security regression) → setelah H1-01..04 selesai
```

Tidak ada dependency keras antar-EPIC H1/H2/H3 — boleh dikerjakan oleh kontributor berbeda secara paralel. Hanya H2-04 dan H3-04 punya dependency internal di dalam EPIC-nya sendiri.

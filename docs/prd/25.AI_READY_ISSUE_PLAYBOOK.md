## 1. Tujuan

Setiap unit kerja hardening menjadi GitHub issue yang bisa diselesaikan **tanpa membaca seluruh PRD**. Issue adalah satuan eksekusi — AI model memuat skill yang relevan, membaca issue + 1–2 section dokumen yang dikutip, lalu eksekusi.

## 2. Prinsip Issue Siap-AI

1. **Self-contained.** Embed fakta yang dibutuhkan; jangan paksa model membaca 10 dokumen.
2. **Context Capsule di atas.** Setiap issue punya kapsul invariant yang sudah diverifikasi terhadap kode (bukan asumsi).
3. **Kutip maksimal 1–2 section + 1 skill.** Cantumkan "read ONLY these".
4. **Atomic.** 1 issue = 1 item EPIC-H1/H2/H3. Pecah jika lebih besar.
5. **AC testable + DoD eksplisit.** Termasuk audit, test regresi keamanan jika relevan.
6. **Pakai skill** `skills/sikesra-*` sesuai bagian.
7. **Label + milestone** yang tepat (`epic:hardening-authz` / `epic:type-consistency` / `epic:test-coverage`).

## 4. Invariant Inti (Terverifikasi Terhadap Kode)

```text
1. Plugin NATIVE: dir awcmsmicro-dev/packages/plugins/awcms-sikesra/
   npm @ahliweb/awcms-sikesra, ID NYATA awcms-sikesra (BUKAN awcms-micro-sikesra)

2. Storage: ctx.storage.<collectionName> — PluginStorageConfig collections
   BUKAN tabel SQL, BUKAN sikesra_* table prefix

3. Audit WAJIB setiap mutasi — pakai createAuditRecord()+appendAuditEvent()
   dari src/runtime.ts (BUKAN src/audit.ts, yang dead code)

4. Verification stages (9, urut):
   draft → submitted_village → verified_village →
   submitted_district → verified_district →
   submitted_sopd → verified_sopd →
   submitted_regency → active_verified

5. DUA union type level yang TIDAK SAMA (lihat H2-01):
   VerificationUserLevel: desa_kelurahan < kecamatan < sopd < kabupaten < admin_sikesra
   VerificationLevel    : desa_kelurahan < kecamatan < sopd < kabupaten_admin < tampil

6. KRITIS: tidak ada route mutasi yang punya otorisasi server-side nyata
   saat ini (lihat 10.SECURITY_AND_PRIVACY_CHECKLIST.md §0). Jangan
   menambah fitur baru di atas fondasi ini tanpa menyadari gap tersebut.

7. Semua string UI via Lingui t`...` atau <Trans>
   Kumo components only (tidak ada HTML native button/input/dialog)
   Logical Tailwind (ms-/me-/ps-/pe-/start-/end-)
```

## 6. Contoh Issue Siap-Eksekusi

### Issue: H1-01 Identitas Terverifikasi pada Route Mutasi

```markdown
## Context Capsule

**Plugin**: @ahliweb/awcms-sikesra (ID nyata: awcms-sikesra)
**Dir**: awcmsmicro-dev/packages/plugins/awcms-sikesra/
**Invariants** (lihat docs/prd/01.AI_IMPLEMENTATION_PROMPT.md §2 HR-07):
- getRequestUserId() saat ini membaca header X-Sikesra-User-Id yang TIDAK
  ditandatangani — ini harus diganti sumber identitas dari sesi EmDash
  yang terverifikasi.
- verification/advance dan verification/reject membaca verifierLevel/actor
  dari body request pemanggil — harus diganti membaca dari identitas sesi.

**Load skill FIRST**: skills/sikesra-api-rbac/SKILL.md

---

## Goal

[H1-01] Ganti sumber identitas pemanggil dari header/body request tak-terverifikasi
menjadi identitas sesi EmDash yang sah, di seluruh route plugin SIKESRA yang
membutuhkan identitas (verification/advance, verification/reject, dan
getRequestUserId()).

---

## References — Read ONLY these

1. `docs/prd/10.SECURITY_AND_PRIVACY_CHECKLIST.md` — §0 (temuan kritis lengkap)
2. `docs/prd/05.API_CONTRACT.md` — §3 (catatan otorisasi per route)
3. Load: `skills/sikesra-api-rbac/SKILL.md`

---

## Scope

**Files to change**:
- `awcmsmicro-dev/packages/plugins/awcms-sikesra/src/runtime.ts` — `getRequestUserId()`,
  `verificationAdvanceRoute`, `verificationRejectRoute`
- `awcmsmicro-dev/packages/plugins/awcms-sikesra/tests/plugin.test.ts` — test baru

**Do NOT touch**: halaman admin (`admin.tsx`), collection storage lain di luar yang
disebut, core EmDash.

---

## Acceptance Criteria

- [ ] AC-1: identitas pemanggil diturunkan dari sesi EmDash yang terverifikasi,
      tidak lagi dari header `X-Sikesra-User-Id` mentah
- [ ] AC-2: request tanpa sesi sah ke route yang butuh identitas ditolak
- [ ] AC-3: `verifierLevel` pada `verification/advance`/`verification/reject`
      tidak lagi dipercaya langsung dari body request

---

## Definition of Done

- [ ] `pnpm --filter @ahliweb/awcms-sikesra typecheck` → 0 error
- [ ] `pnpm --filter @ahliweb/awcms-sikesra test` → pass, termasuk test baru AC-2
- [ ] `pnpm --silent lint:quick` → clean
- [ ] Changeset ditambahkan (perilaku publik plugin berubah)
```

# AI-Ready Issue Playbook & Coverage Index — SIKESRA

**Versi**: 1.0  
**Tanggal**: Juni 2026  
**Fungsi**: Standar penulisan GitHub issue yang self-contained untuk dieksekusi AI model junior bertoken terbatas.

---

## 1. Tujuan

Setiap unit kerja MVP menjadi GitHub issue yang bisa diselesaikan **tanpa membaca seluruh PRD**. Issue adalah satuan eksekusi — AI model memuat skill yang relevan, membaca issue + 1–2 section dokumen yang dikutip, lalu eksekusi.

---

## 2. Prinsip Issue Siap-AI

1. **Self-contained.** Embed fakta yang dibutuhkan; jangan paksa model membaca 10 dokumen.
2. **Context Capsule di atas.** Setiap issue punya kapsul invariant.
3. **Kutip maksimal 1–2 section + 1 skill.** Cantumkan "read ONLY these".
4. **Atomic.** 1 issue = 1 unit kohesif (≤ 1–2 hari kerja). Pecah jika lebih besar.
5. **AC testable + DoD eksplisit.** Termasuk audit, RBAC/ABAC, test, lint.
6. **Pakai skill** `skills/sikesra-*` sesuai bagian.
7. **Label + milestone** yang tepat.

## 4. Invariant Inti

```
1. Plugin NATIVE: dir awcmsmicro-dev/packages/plugins/awcms-sikesra/
   npm @ahliweb/awcms-sikesra, ID awcms-micro-sikesra

2. D1 storage prefix: sikesra_*
   KV keys: sikesra:*
   R2 keys: sikesra/*

3. API prefix: /_emdash/api/plugins/awcms-sikesra/

4. Response envelope:
   List: { data: { items, nextCursor? }, meta }
   Single: { data, meta }
   Error: { error: { code, message }, meta }

5. Audit WAJIB setiap mutasi:
   import { createAuditRecord } from "./audit.js"
   await ctx.storage.auditEvents.create(createAuditRecord({ kind, scope, actor, summary }))

6. Verification stages (urutan):
   draft → submitted_village → verified_village →
   submitted_district → verified_district →
   submitted_sopd → verified_sopd →
   submitted_regency → active_verified

7. User levels: desa_kelurahan < kecamatan < sopd < kabupaten < admin_sikesra

8. Semua string UI via Lingui t`...` atau <Trans>
   Kumo components only (tidak ada HTML native button/input/dialog)
   Logical Tailwind (ms-/me-/ps-/pe-/start-/end-)
```

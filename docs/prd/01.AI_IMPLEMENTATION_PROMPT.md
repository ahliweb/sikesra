## 2. Aturan Keras (Hard Rules — Tidak Boleh Dilanggar)

### HR-01: Plugin-Owned Storage — Jangan Sentuh Core Tables

```typescript
// ❌ SALAH — akses tabel core EmDash
const user = await db.selectFrom("_emdash_users").where("id", "=", userId).executeTakeFirst()

// ✅ BENAR — gunakan ctx.locals.user dari EmDash session
const user = locals.user
```

### HR-02: Jangan Modifikasi Core AWCMS-Micro

```typescript
// ❌ SALAH — modifikasi core auth
// awcmsmicro-dev/packages/core/src/auth.ts (JANGAN DIUBAH)

// ✅ BENAR — extend via plugin hooks
// awcmsmicro-dev/packages/plugins/awcms-sikesra/src/runtime.ts -> createSharedHooks()
```

### HR-03: Semua Mutasi WAJIB Tulis Audit Event

```typescript
// ✅ BENAR — setelah setiap create/update/delete/verify
import { createAuditRecord } from "./audit.js"
const record = createAuditRecord({
  kind: "registry.created",
  scope: "sikesra:registry",
  actor: user?.id ?? "system",
  summary: `Entitas ${entity.label} dibuat`,
  metadata: { entityId: entity.id, entityType: entity.entity_type }
})
await ctx.storage.auditEvents.create(record)
```

### HR-04: Response Envelope Wajib Konsisten

```typescript
// List endpoint
return Response.json({ data: { items, nextCursor }, meta: { requestId, timestamp } })

// Single endpoint
return Response.json({ data: entity, meta: { requestId, timestamp } })

// Error endpoint
return Response.json({ error: { code: "NOT_FOUND", message: "Entitas tidak ditemukan" }, meta: { requestId, timestamp } }, { status: 404 })
```

### HR-05: ABAC Filter Wajib Diterapkan Sebelum Query

```typescript
// ✅ BENAR — selalu filter berdasarkan scope wilayah user
const userLevel = user.attributes?.level as SikesraUserLevel
const regionFilter = getRegionFilter(userLevel, user.attributes)

let query = ctx.db.selectFrom("sikesra_registry_entities")
if (regionFilter.village_code) {
  query = query.where("village_code", "=", regionFilter.village_code)
} else if (regionFilter.district_code) {
  query = query.where("district_code", "=", regionFilter.district_code)
}
```

### HR-06: Jangan Gunakan String Interpolasi di SQL

```typescript
// ❌ SALAH — SQL injection risk
const result = await sql.raw(`SELECT * FROM ${tableName}`).execute(db)

// ✅ BENAR — gunakan Kysely sql.ref() atau parameterized
const result = await ctx.db.selectFrom(sql.ref(tableName) as any).execute()
// Atau: validateIdentifier(tableName) terlebih dahulu
```

## 4. Invariant Inti (selalu berlaku)

```
1. Plugin NATIVE: dir awcmsmicro-dev/packages/plugins/awcms-sikesra/
   npm: @ahliweb/awcms-sikesra
   plugin ID: awcms-micro-sikesra

2. Storage PREFIX: semua tabel plugin: sikesra_*
   ctx.storage.* untuk akses (bukan raw SQL di handler)

3. Verification stages (urutan):
   draft → submitted_village → verified_village →
   submitted_district → verified_district →
   submitted_sopd → verified_sopd →
   submitted_regency → active_verified

4. User levels (hierarki):
   desa_kelurahan < kecamatan < sopd < kabupaten < admin_sikesra

5. Sensitivity levels (akses semakin ketat):
   public_safe < internal < restricted < highly_restricted

6. Audit: WAJIB setiap mutasi. kind format: "domain.action"
   e.g. "registry.created", "verification.approved", "access.role_assigned"

7. Semua string user-facing via Lingui: t`...` atau <Trans>
   Locale default: en. Supported: en, id.

8. Admin UI: Kumo components ONLY. Tidak boleh custom button/input/dialog.
   Check: `npx @cloudflare/kumo doc Button`

9. Tailwind: logical classes ONLY (ms-*, me-*, ps-*, pe-*, start-*, end-*)
   Tidak boleh ml-*, mr-*, pl-*, pr-*, left-*, right-*

10. Test: setiap bug harus punya failing test sebelum fix.
    Framework: vitest. Real storage (bukan mock).
```

## 7. Pola yang Sering Dipakai

### Membuat route handler baru di runtime.ts

```typescript
// Di createNativeRoutes() dalam runtime.ts
{
  path: "/_emdash/api/plugins/awcms-sikesra/registry",
  method: "GET",
  handler: async (request, ctx) => {
    // 1. Auth check
    const user = ctx.locals.user
    if (!user) return new Response(null, { status: 401 })

    // 2. Permission check
    const hasPerm = ctx.plugin.hasPermission(user, AWCMS_SIKESRA_PERMISSIONS.dashboardRead)
    if (!hasPerm) return new Response(null, { status: 403 })

    // 3. ABAC filter
    const regionFilter = buildRegionFilter(user)

    // 4. Query
    const items = await ctx.storage.registryEntities.findMany({ ...regionFilter, limit: 50 })

    // 5. Return envelope
    return Response.json({ data: { items, nextCursor: undefined }, meta: buildMeta() })
  }
}
```

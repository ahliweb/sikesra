# API Contract — SIKESRA (39 Route Nyata)

**Sumber**: `sharedRouteEntries` di `src/runtime.ts`, diekspos lewat `createNativeRoutes()` (produksi, dipanggil dari `createPlugin()` di `src/index.ts`) dan `createSandboxRoutes()` (mode sandbox).

> Path di bawah adalah nama route native plugin (bukan path REST konvensional `/registry/:id`). Hanya `public/status` berflag `public: true` — 38 route lain tidak punya flag publik (lihat catatan otorisasi di § 3).

## 1. Halaman Admin (Manifest)

16 halaman terdaftar di `AWCMS_SIKESRA_MANIFEST`: `overview`, `registry`, `documents`, `import`, `verification`, `audit`, `reports`, `access/permissions`, `access/roles`, `access/matrix`, `access/preview`, `abac/attributes`, `abac/policies`, `abac/preview`, `regions`, `data-types`. Plus 3 widget (`GovernanceWidget`, `AccessRightsHealthWidget`, `AbacPolicyStatusWidget`) dan 1 field widget (`StatusBadgeField`).

## 3. Catatan Otorisasi (Kritis — Baca Sebelum Memakai Endpoint Ini sebagai Contoh)

Grep penuh terhadap `ctx.user`, pengecekan permission, atau guard otorisasi di badan handler route: **tidak ditemukan satu pun** untuk route mutasi (`*/save`, `verification/advance`, `verification/reject`, `import/promote`). Satu-satunya gerbang kondisional yang ada adalah pengecekan level verifier di `verification/advance`/`verification/reject`:

```typescript
const allowedVerifierLevels = getAllowedVerifierLevels(item.currentLevel);
if (!allowedVerifierLevels.includes(verifierLevel)) { /* tolak */ }
```

tetapi `verifierLevel` dibaca langsung dari body request milik caller (`getString(routeCtx.input, "verifierLevel")`), dengan fallback ke `inferVerifierLevel(actor)` di mana `actor` juga dikirim caller — **bukan diturunkan dari sesi/identitas yang terverifikasi**. Ini adalah pengecekan business-rule yang percaya klaim sendiri (self-asserted), bukan otorisasi nyata. `getRequestUserId()` membaca header `X-Sikesra-User-Id` yang tidak ditandatangani, hanya dipakai untuk filter tampilan di `verification/list`, tidak pernah sebagai gerbang mutasi.

`evaluateAbacDecision()`/`previewAccess()` hanya dipanggil dari route bertanda "preview"/"demo" (#27, #37, #38) — **tidak pernah menggerbang mutasi nyata** di route manapun.

Lihat `10.SECURITY_AND_PRIVACY_CHECKLIST.md` untuk dampak dan rekomendasi hardening — ini adalah gap yang harus ditangani sebelum plugin ini dipakai untuk data produksi sungguhan.

## 4. Request/Response Body — Route Mutasi Kritis

Semua route mengembalikan envelope `{ "data": <payload> }` saat sukses, `{ "error": { "code": "...", "message": "..." } }` saat gagal.

### `registry/save`

```jsonc
// Request body
{
  "id": "ulid-string",              // wajib untuk update; kosong/omit untuk create
  "label": "Masjid Al-Falah",       // wajib
  "entityType": "rumah_ibadah",     // wajib; enum 8 nilai (lihat fixtures.ts DEFAULT_DATA_TYPES)
  "provinceCode": "62",             // wajib
  "regencyCode": "6205",            // wajib
  "districtCode": "620504",         // wajib
  "villageCode": "6205042001",      // wajib
  "publicSummary": "...",           // opsional; ringkasan aman tanpa PII
  "sensitivity": "internal",        // opsional; default per entity_type (lihat 17.DATA_GOVERNANCE_AND_RETENTION_POLICY.md §2.1)
  "metadata": {}                    // opsional; field tambahan bebas
}
// Response: { "data": { entity: SikesraReferenceRegistryEntity } }
// Catatan: stage selalu diset ke "submitted_village" oleh server, bukan dari body
```

### `verification/advance`

```jsonc
// Request body
{
  "entityId": "ulid-string",        // wajib
  "verifierLevel": "kecamatan",     // wajib; VerificationUserLevel enum
  "actor": "username-or-id",        // wajib (saat ini self-asserted — akan digantikan oleh H1-01)
  "notes": "..."                    // opsional
}
// Response: { "data": { entity: SikesraReferenceRegistryEntity, event: SikesraReferenceVerificationEvent } }
// Error: { "error": { "code": "SIKESRA_LEVEL_MISMATCH", "message": "..." } } jika verifierLevel tidak sesuai stage saat ini
```

### `verification/reject`

```jsonc
// Request body
{
  "entityId": "ulid-string",        // wajib
  "verifierLevel": "kecamatan",     // wajib
  "actor": "username-or-id",        // wajib
  "notes": "Alasan penolakan..."    // wajib (divalidasi non-empty untuk reject)
}
// Response: { "data": { entity: SikesraReferenceRegistryEntity, event: SikesraReferenceVerificationEvent } }
// Stage kembali ke "needs_review" pada revision target (bukan ke "draft")
```

### `documents/save`

```jsonc
// Request body
{
  "id": "ulid-string",              // opsional; kosong untuk create baru
  "entityId": "ulid-string",        // wajib; ID entity registry yang dikaitkan
  "documentType": "ktp",            // wajib; enum bebas (belum ada constraint di kode saat ini)
  "sensitivity": "restricted",      // wajib
  "label": "KTP Ketua",             // wajib
  "metadata": {}                    // opsional
}
// Response: { "data": { document: SikesraReferenceSupportingDocument } }
// Catatan: TIDAK ada upload file — ini hanya metadata
```

### `import/promote`

```jsonc
// Request body
{
  "rows": [
    {
      "label": "Masjid Al-Falah",
      "entityType": "rumah_ibadah",
      "provinceCode": "62",
      "regencyCode": "6205",
      "districtCode": "620504",
      "villageCode": "6205042001"
    }
    // ... lebih banyak baris
  ]
}
// Response: { "data": { promoted: number, entities: SikesraReferenceRegistryEntity[] } }
// Semua baris diset ke stage "submitted_village"
```

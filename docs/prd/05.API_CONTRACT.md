# API Service Contract — SIKESRA v1.0

**Base URL**: `/_emdash/api/plugins/awcms-sikesra`  
**Auth**: EmDash session (cookie)  
**CSRF**: Header `X-EmDash-Request: 1` wajib untuk state-changing requests  
**Format**: JSON (Content-Type: application/json)  

---

## 1. Response Envelope

### Success (single)
```json
{
  "data": { "...entity..." },
  "meta": { "requestId": "uuid", "timestamp": "ISO8601" }
}
```

### Success (list)
```json
{
  "data": {
    "items": [{ "...entity..." }],
    "nextCursor": "base64-cursor-or-null"
  },
  "meta": { "requestId": "uuid", "timestamp": "ISO8601" }
}
```

### Error
```json
{
  "error": { "code": "SIKESRA_ENTITY_NOT_FOUND", "message": "Entitas tidak ditemukan" },
  "meta": { "requestId": "uuid", "timestamp": "ISO8601" }
}
```

## 4. Registry Endpoints

### GET /registry

**Permission**: `awcms:sikesra:registry:read`  
**ABAC**: Filter otomatis berdasarkan level user

**Query Params**:
| Param | Type | Default | Deskripsi |
|-------|------|---------|-----------|
| `cursor` | string | - | Cursor paginasi |
| `limit` | number | 50 | Max items (max: 100) |
| `entityType` | string | - | Filter tipe (comma-separated) |
| `stage` | string | - | Filter stage |
| `districtCode` | string | - | Filter kecamatan |
| `villageCode` | string | - | Filter desa |
| `sensitivity` | string | - | Filter sensitivity |
| `q` | string | - | Search di label/code |

**Response 200**:
```json
{
  "data": {
    "items": [
      {
        "id": "ulid",
        "code": "RI-001",
        "label": "Masjid Al-Ikhlas",
        "entityType": "rumah_ibadah",
        "sensitivity": "internal",
        "region": {
          "provinceCode": "62",
          "regencyCode": "6201",
          "districtCode": "620101",
          "villageCode": "6201010001"
        },
        "verificationStage": "verified_district",
        "inputLevel": "desa_kelurahan",

        "createdAt": "ISO8601",
```

### POST /registry

**Permission**: `awcms:sikesra:registry:write`  
**Headers**: `X-EmDash-Request: 1`

**Request Body**:
```json
{
  "label": "Masjid Al-Ikhlas",
  "entityType": "rumah_ibadah",
  "sensitivity": "internal",
  "region": {
    "provinceCode": "62",
    "regencyCode": "6201",
    "districtCode": "620101",
    "villageCode": "6201010001"
  },
  "publicSummary": "Masjid di Kelurahan Baru",
  "extraData": {}
}
```

**Response 201**:
```json
{
  "data": { "...entity..." },
  "meta": { "..." }
}
```

**Errors**:
- `400 VALIDATION_ERROR`: field wajib tidak ada
- `403 SIKESRA_REGION_FORBIDDEN`: user tidak bisa input di wilayah ini

### GET /registry/:id

**Permission**: `awcms:sikesra:registry:read`

---

### PUT /registry/:id

**Permission**: `awcms:sikesra:registry:write`  
**Headers**: `X-EmDash-Request: 1`  
**Note**: Hanya bisa update jika stage masih `draft` atau `needs_review`

---

### DELETE /registry/:id

**Permission**: `awcms:sikesra:registry:write`  
**Headers**: `X-EmDash-Request: 1`  
**Note**: Soft delete — set sensitivity ke "archived", tidak hapus fisik

---

### POST /registry/:id/verify

**Permission**: `awcms:sikesra:verification:verify`  
**Headers**: `X-EmDash-Request: 1`

**Request Body**:
```json
{
  "result": "approved | needs_review | rejected",
  "notes": "Dokumen lengkap, data valid"
}
```

**Validasi**:
- `notes` wajib jika `result` = `needs_review` atau `rejected`
- User level harus sesuai stage saat ini
- `desa_kelurahan` → approve `draft` → `submitted_village`
- `kecamatan` → approve `verified_village` → `verified_district`
- dll.

**Response 200**:
```json
{
  "data": {
    "entityId": "ulid",
    "previousStage": "verified_village",
    "newStage": "verified_district",
    "verificationEvent": { "...event..." }
  },
  "meta": { "..." }
}
```

**Errors**:
- `403 SIKESRA_LEVEL_MISMATCH`: user level tidak sesuai untuk stage ini
- `422 SIKESRA_STAGE_INVALID`: transisi stage tidak valid

---

## 5. Documents Endpoints

### GET /documents

**Permission**: `awcms:sikesra:documents:read`  
**Query**: `entityId` (wajib), `sensitivity`

---

### POST /documents

**Permission**: `awcms:sikesra:documents:write`  
**Headers**: `X-EmDash-Request: 1`  
**Content-Type**: `multipart/form-data`

**Form Fields**:
- `file`: binary (max 10MB, PDF/JPG/PNG)
- `entityId`: string (wajib)
- `documentType`: string
- `title`: string
- `sensitivity`: string (default: internal)
- `issuedAt`: string (ISO8601, opsional)

---

### GET /documents/:id/download

**Permission**: `awcms:sikesra:documents:read`  
**Response**: Redirect ke signed R2 URL (5 menit TTL)

## 10. HTTP Status Codes

| Code | Kondisi |
|------|---------|
| 200 | OK |
| 201 | Created |
| 400 | Validation error |
| 401 | Unauthenticated |
| 403 | Forbidden (RBAC/ABAC) |
| 404 | Not found |
| 409 | Conflict (code duplikat) |
| 413 | File terlalu besar |
| 422 | Business rule violation (stage invalid) |
| 500 | Internal server error |

## 11. Verification Stage Transition Table

| Current Stage | User Level | Action | New Stage |
|--------------|------------|--------|-----------|
| `draft` | `desa_kelurahan` | approved | `submitted_village` |
| `submitted_village` | `desa_kelurahan` / `kecamatan` | approved | `verified_village` |
| `verified_village` | `kecamatan` | approved | `submitted_district` |
| `submitted_district` | `kecamatan` | approved | `verified_district` |
| `verified_district` | `sopd` / `kecamatan` | approved | `submitted_sopd` |
| `submitted_sopd` | `sopd` | approved | `verified_sopd` |
| `verified_sopd` | `kabupaten` / `sopd` | approved | `submitted_regency` |
| `submitted_regency` | `kabupaten` | approved | `active_verified` |
| any | verifier level | needs_review | prev stage |
| any | verifier level | rejected | `draft` |

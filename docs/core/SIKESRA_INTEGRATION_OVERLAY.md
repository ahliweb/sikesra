# SIKESRA Integration Overlay for Core Docs

Status: Active for this repository (`ahliweb/sikesra`).

This overlay reconciles the generic 15-part AWCMS-Micro core documentation with the current self-contained SIKESRA deployment. The core docs remain architectural guidance; this file is the repository-specific bridge that explains how each core rule is applied here without turning this repository into a fragile EmDash fork.

## Repository Boundary

This repository is not the reusable `awcms-micro-standard` scaffold described in the generic core docs. It is the SIKESRA deployment target.

| Repository Concept | Generic Core Docs | This Repository |
|---|---|---|
| Upstream authority | `emdash-cms/emdash` | Still authoritative for EmDash behavior. |
| AWCMS reference host | `ahliweb/awcms-micro` / standard scaffold | Reference only; do not deploy SIKESRA from there. |
| Runtime/deploy repo | Client-specific repo | This repository: `ahliweb/sikesra`. |
| Plugin package path | Often shown as `packages/plugins/<module>` | Current SIKESRA runtime source is local `src/`. |
| Deployment entry | EmDash/Astro Worker output | Hybrid wrapper: `dist/server/worker-wrapper.mjs` generated from `scripts/worker-wrapper-template.mjs`. |

## Build and Deploy Map

Generic core docs often show `pnpm` and scaffold-style Wrangler examples. This repository uses the following actual pipeline:

```bash
npm run typecheck
npm test
npm run build
set -a && source .env && set +a && npx wrangler deploy
```

`npm run build` runs:

```bash
astro build && node scripts/postbuild.mjs
```

The postbuild script is part of this repository's integration contract and must remain minimal:

1. Sets `dist/server/wrangler.json` `main` to `worker-wrapper.mjs`.
2. Strips `import "cloudflare:workers"` from `dist/server/entry.mjs` for hybrid wrapper compatibility.
3. Injects the SIKESRA public HTML into the worker wrapper template.
4. Does not patch EmDash source, `node_modules`, or generated EmDash admin chunks.

## Admin Integration Contract

Target contract for the future rebuilt SIKESRA admin API: the EmDash admin client reads plugin Block Kit responses as:

```txt
(await response.json()).data.blocks
```

Therefore `/_emdash/api/plugins/sikesra/admin` must return:

```json
{
  "data": {
    "blocks": []
  }
}
```

Returning a raw `{ "blocks": [] }` payload breaks the admin page with `Cannot read properties of undefined (reading 'blocks')`.

During the scratch rebuild, unfinished SIKESRA APIs remain disabled with `503`, but the admin Block Kit route `/_emdash/api/plugins/sikesra/admin` and public-safe route group `/_emdash/api/plugins/sikesra/public/*` are allowed through to EmDash so the plugin shell and `/sikesra` public page can render. If future implementation requires wrapper-owned admin rendering because the native EmDash plugin route context does not expose raw Cloudflare bindings such as `env.SIKESRA_DB`, create a separate adapter decision before implementing it.

The previously used workaround pattern must not be restored without that decision:

1. Wrapper receives `/_emdash/api/plugins/sikesra/admin`.
2. Wrapper calls EmDash first with `request.clone()`.
3. If EmDash denies auth/route/permission, return that response.
4. If allowed, wrapper renders SIKESRA Block Kit using `env.SIKESRA_DB`.
5. Wrapper returns the EmDash plugin API envelope.

## Activation Contract

Plugin activation in `_plugin_state` currently controls only the SIKESRA public placeholder while admin/API behavior is disabled for rebuild:

| Plugin State | `/sikesra` | `/_emdash/api/plugins/sikesra/*` |
|---|---|---|
| active or missing state | Enabled | `admin` and `public/*` enabled; unfinished routes return `503` rebuild placeholder |
| inactive | `404` | `admin` and `public/*` enabled; unfinished routes return `503` rebuild placeholder |

```bash
node --check scripts/worker-wrapper-template.mjs
```

After deploy, validate:

```txt
/                                      -> 200, x-route: emdash-root
/sikesra                               -> 200, x-route: sikesra
/_emdash/admin                         -> EmDash auth/admin shell
/_emdash/api/plugins/sikesra/admin     -> authenticated EmDash Block Kit response
/_emdash/api/plugins/sikesra/public/*  -> public-safe aggregate responses
/_emdash/api/plugins/sikesra/v1/*      -> 503 until rebuilt
```

## Known Integration Gaps

These are not contradictions in the core docs; they are local implementation gaps or adapters:

1. Shared AWCMS permission registry helper is not yet available; SIKESRA uses a local permission catalog.
2. Shared ABAC/audit adapters are not yet available; SIKESRA uses local D1-backed fallbacks.
3. Full React admin pages are not implemented yet; current admin pages use EmDash Block Kit payloads.
4. Admin Block Kit integration must be rebuilt and verified with a real admin session after API security is implemented.
